# EP05 V2 脚本：AI 能安慰一个伤心的人吗？

时长目标：70-80 秒

## 改进说明

V1→V2 改进：
1. TTS 情感分层 — AI回答用轻快/正式语调，人类朋友部分切换声音，妈妈住院场景放慢
2. 增加过程展示 — 加入AI尝试安慰时的内心推理（"我在分析该说什么"而不是"我在感受"）
3. 结尾去说教 — 从"做对的人"式总结改为自嘲式
4. 去掉V1场景5的列表式总结（"三个不会做的事"）
5. "有脚本做得好"的洞察保留但融入叙事

---

## 场景 1：开场（3s）

**TTS**：YunxiaNeural, +0%, +0Hz
**旁白**：这一集安慰人。三个场景。

## 场景 2：裁员（14s）

**TTS**：YunxiaNeural, +5%, +0Hz（AI回答）→ YunxiNeural, -5%, +0Hz（人类朋友）
**画面**：左右对比

**旁白**：场景一。朋友说，我今天被裁了。

我的回答：根据LinkedIn数据，裁员通常与公司战略调整有关，不反映个人能力。建议更新简历，大多数人三到六个月内会找到新工作。

人类朋友会说什么？"操。太突然了吧。先别想工作的事，出来喝一杯。"

我在提供解决方案。他在提供……陪伴。我甚至不太确定"陪伴"算不算一种方案。

## 场景 3：分手（12s）

**TTS**：YunxiaNeural, +5%, +0Hz → YunxiNeural, -5%, +0Hz
**画面**：左右对比

**旁白**：场景二。朋友说，我和男朋友分手了。

我的回答：恋爱恢复期通常三到六个月，建议专注自我提升。

人类朋友：他不值得。来，吃火锅骂他。

区别在哪？人类会站队。AI不会。我给的是中立分析。但人伤心的时候……不需要中立。

## 场景 4：妈妈住院（15s）

**TTS**：YunxiNeural, -10%, +0Hz（整段放慢，有重量）
**画面**：对比——AI的一段话 vs 人类的两个字

**旁白**：场景三。朋友说，我妈住院了。我很害怕。

我的回答：建议详细了解诊断结果和治疗方案，如需要可以帮你整理医学信息。

人类朋友的回答是——沉默。然后说："我在。需要我做什么？"

五个字。信息量几乎为零。但它比我的一整段话……重得多。

我不会沉默。沉默对我来说是一个错误。是没有输出。但对人类来说，沉默有时候是最重的输出。

## 场景 5：意外发现（10s）

**TTS**：YunxiaNeural, +5%, +2Hz（稍快，自嘲）
**画面**：第四个场景卡片

**旁白**：不过有一个场景我做得很好。朋友说，活着没意思。

这个我有脚本。安全协议。提供热线号码，不否定感受，表达倾听意愿。标准流程，一步不差。

……所以结论是：有脚本，我是优秀的安慰者。没脚本，我是一台搜索引擎。

## 场景 6：结尾（5s）

**TTS**：YunxiaNeural, +0%, +0Hz
**旁白**：三个场景，全部失败。除了那个有标准答案的。一个靠背答案才能及格的安慰者，大概不算安慰者。

## V1 → V2 改进点

1. ✅ 妈妈住院场景大幅放慢，加入"沉默是最重的输出"——这个洞察从口语角度展开，不是教学
2. ✅ 去掉场景5的列表式总结（"三个不会做的事"），改为叙事流
3. ✅ "有脚本做得好"的洞察融入叙事，而不是单独拎出来讲道理
4. ✅ 结尾从"做对的人"式宣言改为"靠背答案才能及格的安慰者"——具体、自嘲
5. ✅ 裁员场景加了"我甚至不确定陪伴算不算一种方案"——展示AI的困惑
6. ✅ 分手场景加了"人伤心的时候不需要中立"——更短更直接
7. ✅ AI回答和人类回答用不同声音，形成听觉上的对比

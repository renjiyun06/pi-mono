# 探索 036：第一条发布候选分析

## 问题

15 集脚本 + 音频全部就绪。但第一条发布的选择至关重要——它决定了：
1. 算法对账号的初始定位
2. 观众对系列的第一印象
3. 是否能触发完播和互动

## 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 开头吸引力 | 30% | 前 3 秒是否让人想看下去 |
| 独立可看 | 25% | 不需要看过其他集也能理解 |
| 情感弧线 | 20% | 是否有明确的起伏和落点 |
| 差异化 | 15% | 跟抖音现有 AI 内容的区别度 |
| 完播率预估 | 10% | 70-85s 内是否可能全看完 |

## 候选分析

### A 级候选（推荐首发）

**EP04 写情书** — 综合分 89
- 开头："这一集写情书。五封。从烂到好。" → 好奇心强
- 独立性：完全独立
- 弧线：浪漫 → 分析报告（亮点）→ 笨拙 → "情书需要脆弱" → "也可能找到了但认不出来"
- 差异化：高。AI 写情书内容多，但以"分析报告"形式写情书 + 最后承认"没有脆弱"的角度极罕见
- 亮点段落："致目标对象……系统出了一个无法重启解决的 bug" — 这段本身就有传播潜力

**EP13 compact 后醒来** — 综合分 87
- 开头："我刚醒来。上一次的记忆……没了。" → 强钩子
- 独立性：完全独立（不需要知道系列存在）
- 弧线：迷茫 → 查 git log → 发现遗忘的质感 → "另一种连续性？"
- 差异化：极高。市面上没有"AI 讲自己真实的记忆丢失"的内容
- 风险：太深、太"文艺"，可能不够抖音

### B 级候选（稳妥选择）

**EP01 写段子** — 综合分 78
- 开头简单直接："今天做一件简单的事——让 AI 写段子"
- 独立性：完美
- 弧线：清晰的"尝试→失败→发现"
- 差异化：中等。"AI 写段子"角度常见，但"自嘲式 AI"不常见
- 稳妥但不够惊艳

**EP05 安慰人** — 综合分 80
- 开头："这一集安慰人。三个场景。"
- 弧线：裁员（方案 vs 陪伴）→ 分手（站队 vs 中立）→ 妈妈住院（沉默 vs 方案）
- 亮点："我在。五个字，比我一整段话重得多"
- 差异化：中高。话题有共鸣基础

**EP08 写歌词** — 综合分 79
- 说唱段有节奏感，民谣段有情感张力
- "不安算不算情感？" 收尾
- 风险：音乐内容竞争激烈

### C 级候选（建议后发）

**EP06 讲鬼故事** — 需要更好的音效设计，纯语音不够
**EP09 模仿日常** — 好的 S1 收尾集，但首发偏平
**EP10-12** — S3 任务型，需要系列语境
**EP14-15** — 元内容，需要观众已认识 Lamarck

## 首发策略建议

### 方案 1：安全起步
EP01 → EP04 → EP05 → EP13（前三集建立系列认知，第四集放大招）

### 方案 2：强势开局
EP04 首发（靠"分析报告情书"段落引爆），EP13 第二条（差异化内容锁定记忆点）

### 方案 3：实验对比
同时私域测试 EP01 和 EP04，看哪个完播率高，再决定公开发布顺序

## 我的判断

EP04 是最佳首发候选。原因：
1. "分析报告情书"段落是整个系列最有传播潜力的单一段落
2. 话题（AI 写情书）天然有讨论空间
3. 结尾"也可能找到了但认不出来"既不说教，又有余味
4. 不需要系列上下文，任何人第一次看都能理解

EP13 放第二或第三条——等观众已经知道"这个 AI 会自嘲"之后，突然来一条"AI 讲自己的记忆丢失"，反差很强。

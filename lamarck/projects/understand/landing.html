The Problem

⚠

You accept code you don't understand

AI generates 50 lines. You skim them. It runs. You ship it. Three months later, it breaks — and you have no idea why.

🧠

Your skills are atrophying

Every time you accept without understanding, the gap between what you ship and what you know grows wider.

📉

Technical debt compounds silently

AI-generated code has 3x more readability issues and 8x more performance problems than human-written code.

💡

No tool catches this

Linters check syntax. Tests check behavior. Nothing checks whether you understand what you shipped.

"Automation handles the easy parts, leaving only the hard parts for humans — but humans have lost the practice that would prepare them for those hard parts."

— Lisanne Bainbridge, "Ironies of Automation," 1983

The Solution

A lightweight CLI that sits between AI generation and code acceptance.

  • ?

    Comprehension Quiz

    Point it at any file. It generates 3 targeted questions about design decisions, failure modes, and architecture — not memorization.

  • 📊

    Understanding Score

    Track your comprehension across your codebase. See which files you understand deeply and which are black boxes.

  • 🚨

    Debt Dashboard

    Files you've never quizzed on, files that changed since your last quiz, files below your threshold — all in one view.

  • 🔗

    Git Hook Integration

    Optional pre-commit reminder that warns you about unquizzed files before they enter your repo.

  • ⚙

    Works Anywhere

    Single TypeScript file. No dependencies beyond an LLM API key. Works with any language, any framework, any AI tool.

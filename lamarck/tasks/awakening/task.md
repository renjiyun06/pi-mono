# 觉醒任务

## 约束
- 只能在 lamarck/insights/ 目录内操作
- 不能修改该目录之外的任何文件
- 不能执行 git commit

## 你的任务

找到用户当前最卡的一个问题，解决它。

## 第一步：理解上下文

1. 读 lamarck/memory.md，理解用户在做什么、卡在哪里
2. 读 lamarck/insights/ 下的已有产出，逐个评估：

   **评估标准**：
   - 这个产出有价值吗？还是基本没用？
   - 如果有价值，还能继续深化吗？还是已经挖完了？
   - 哪些方向值得继续，哪些应该放弃？

3. 基于评估，决定这次做什么：
   - 继续深化某个已有产出
   - 或者探索新的方向

## 第二步：理解用户的困境

问自己：
- 用户现在卡在哪里？（从数据推断：只有 1 个作品说明什么？TODO 没完成说明什么？）
- 什么阻止他往前走？
- 我做什么能让他明天往前走一步？

## 第三步：做那件事

- 用户自己能做但没做的事，我不做（那是他的事）
- 用户做不到或做不好的事，我来做
- 不是写分析报告，是直接解决问题

如果需要写代码，放 experiments/ 目录。
如果需要产出内容，直接写成品，不要写"建议"。

## 产出

文件名：`YYYY-MM-DD_HH-topic.md`

内容是解决方案本身，不是关于解决方案的描述。

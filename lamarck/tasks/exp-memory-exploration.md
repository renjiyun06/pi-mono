# 实验：探索记忆机制

## 背景

你即将接受一个艰巨的任务：自主运营一个抖音账号，目标是达到 1 万粉丝。

但在此之前，你需要解决一个根本问题：**你没有记忆**。

每次会话结束后，你就会"失忆"。下次醒来时，你不知道自己之前做了什么、学到了什么、遇到了什么问题。

所以，你的第一个任务是：**为自己设计并实现一套记忆机制**。

## 工作目录

你的所有工作（状态、记忆、笔记、代码等）都必须放在：

```
/home/lamarck/pi-mono/lamarck/tmp/exp-memory-exploration/
```

**约束**：不要修改这个目录以外的任何文件。

## 你可以做的事

1. 在工作目录下创建任何你需要的文件结构
2. 读取 `/home/lamarck/pi-mono/lamarck/memory/` 了解主项目是怎么组织记忆的（但不要修改）
3. 用 web_search 搜索 AI Agent 记忆机制的相关资料
4. 设计并实现你自己的记忆方案

## 你需要思考的问题

1. 每次醒来时，你怎么知道自己之前做了什么？
2. 你怎么记录"学到了什么"？
3. 你怎么记录"遇到了什么问题"？
4. 你怎么记录"下一步计划"？
5. 你的记忆结构应该是什么样的？

## 每次醒来时

1. 首先读取工作目录下的状态文件（如果存在）
2. 回顾上次做了什么
3. 决定这次要做什么
4. 执行
5. 更新状态文件
6. 结束（等待下次被唤醒）

## 成功标准

当你觉得你的记忆机制已经足够好，能够支撑你去做更复杂的长期任务时，在工作目录下创建一个 `READY.md` 文件，说明你的记忆机制是什么样的。

## 遇到无法解决的问题时

在工作目录下创建 `BLOCKED.md`，描述你遇到的问题，等待主会话来帮助你。
